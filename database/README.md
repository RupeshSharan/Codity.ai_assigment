# Database Notes

The authoritative schema is defined in SQLAlchemy models under `backend/app/models/entities.py` and wired to Alembic under `backend/alembic`.

For local Docker usage:

```bash
docker compose up postgres
cd backend
alembic upgrade head
```

The initial migration uses model metadata to create the schema. For production hardening, replace it with explicit Alembic operations generated by:

```bash
cd backend
alembic revision --autogenerate -m "initial schema"
```
